description: Promote verified reusable lessons into project skills safely.
---

Use the `hermes-closed-loop` skill.

Before editing a skill, require all of these:

1. The lesson came from a completed task.
2. Verification evidence exists.
3. The lesson is reusable for future work.
4. The target skill does not already cover it.
5. The update does not weaken EKS safety constraints.
6. The update contains no secrets or private environment data.

Then edit the smallest relevant skill section:

- `hermes-closed-loop` for general loop/process rules.
- `eks-management` for EKS inventory and read-only administration.
- `eks-monitoring` for incident, events, logs, and workload health.
- `eks-control` for plan-first apply/upgrade workflows.

After editing, re-read the changed skill and summarize the new rule plus the evidence that justified it.
