---
description: Reflect on completed work and decide whether project skills should be updated.
---

Use the `hermes-closed-loop` skill.

Review the just-completed work:

1. Summarize what changed or what was learned.
2. List verification evidence.
3. Decide whether there is a durable, reusable lesson.
4. If yes, choose the target skill and write the smallest safe update.
5. If not yet verified, append a candidate lesson to `.opencode/learning/skill-backlog.md`.
6. If no lesson exists, state `Skill update: not needed`.

Never record secrets, kubeconfig content, Kubernetes Secret data, access tokens, or private incident payloads.
